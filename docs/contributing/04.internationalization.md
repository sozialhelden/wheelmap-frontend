# Internationalization

We use Transifex Native for translations. The app fetches translated strings
directly from Transifex at runtime, so we don't store translations in the
codebase.

## Using translations

Use the Transifex Native **JavaScript SDK** to translate strings in your code:
[Transifex Native JavaScript Docs](https://developers.transifex.com/docs/javascript-sdk-internationalize-code)

For **React components**, use the `@transifex/react` package:
[Transifex React Docs](https://developers.transifex.com/docs/react-sdk#usage)

Both packages are already installed. Prefer the `T` and `UT` **React
components** over the JavaScript SDK when possible—they're more efficient.

## Adding new translation strings

> [!IMPORTANT]
> Make sure `TRANSIFEX_NATIVE_SECRET` and `TRANSIFEX_API_TOKEN` are set in your
> `.env` file.

After adding a new string that needs translation, push it to Transifex. Do this
after merging your PR into main, but before deploying to production.

Run this command to scan and upload all strings:

```bash
pnpm transifex:push
```

## Definition of done for translations

All strings must be translated into English and German before deploying to
production.
